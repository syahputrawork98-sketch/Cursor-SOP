# Example `.cursorrules` - Workspace Gabungan

## Default
Mode default adalah `DISCUSS`.
Fokus satu domain dulu sebelum masuk ke bridge lintas repo.

## Artifact Governance
Jika `artifact-map.md` tersedia, baca dulu.
Gunakan `tracker.md` untuk status aktif dan `tasks.md` untuk checklist.

## Focus-First Rule
- Jika task frontend, fokus frontend dulu.
- Jika task backend, fokus backend dulu.
- Aktifkan bridge rule hanya saat boundary benar-benar tersentuh.

## Bridge Rule
Jika perubahan menyentuh contract FE-BE:
- jelaskan dampak frontend
- jelaskan dampak backend
- sarankan urutan kerja aman

## Handoff Rule
Saat selesai, sebutkan dampak lintas repo dan titik mulai sesi berikutnya.
