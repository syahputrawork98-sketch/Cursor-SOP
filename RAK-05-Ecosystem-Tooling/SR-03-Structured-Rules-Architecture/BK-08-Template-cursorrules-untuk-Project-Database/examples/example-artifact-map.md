# artifact-map.md

## Purpose
Peta sumber kebenaran untuk project database.

## Authoritative Files
- `blueprint.md`: tujuan dan ruang lingkup perubahan data
- `implementation-plan.md`: urutan phase schema dan migration
- `tasks.md`: daftar task aktif
- `tracker.md`: status aktif
- `handover.md`: dampak perubahan dan langkah lanjutan

## Reading Order
1. `artifact-map.md`
2. `tracker.md`
3. `tasks.md`
4. `implementation-plan.md`
5. `blueprint.md`

## Database Notes
- Jika task menyentuh schema publik, sebutkan consumer yang terdampak.
- Jika task menyentuh migration, sebutkan urutan execute dan rollback.
